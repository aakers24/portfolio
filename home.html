Obstruction AI


Description

Obstruction is a game in which each player tries to be the last person to make a move. A move is made by placing your piece in a position on the board. When a position is occupied by a player's piece, none of the adjacent positions on the board can be occupied. So, if it's your turn and there is no place remaining for you to legally place your piece, you lose.

The project is a console application in which an AI plays against a human player in a game of Obstruction. I implemented the project in such a way that with small tweaks the board size and look-ahead can be changed. This means that the board size can theoretically be arbitrarily large, and the look-ahead of the AI can be arbitrarily long. The look-ahead is how many turns the AI looks into the future to make its move.


Implementation

The AI uses a Minimax algorithm to look into the future and see the possible moves that could be made given the current board state. It uses a function to estimate the best move and makes it.

However, as the board size and/or look-ahead distance increases this becomes very computationally expensive and quickly becomes unplayable due to the time it takes for the AI to consider it's moves. Because of this, there is an option to use an Alpha-Beta Pruning Minimax algorithm. This does mostly the same thing as the Minimax algorithm, but avoids unnecessary calculations by determining if an optimal solution has already been found. This greatly increases the usability of the AI as the board size and/or look-ahead distance increases.


Importance

What makes this one of my best works is that I went the extra mile to make the finished product work well and scale easily. It is a fun game and the AI will indeed beat me. Working on this project really helped me understand AI, and working to make it scale to an arbitrary board size was a challenge which really helped me get the most out of the project. It was very rewarding playing against the AI on all the different board sizes and look-ahead levels as well as with and without AB-Pruning because it showed the efficacy of my skills and the optimality of my result.

Recursive Nth Fibonacci Number in Assembly


Description

The Fibonacci Sequence is a pattern of numbers that has various applications in mathematics and other fields. In general the Nth number in the sequence is the sum of the 2 before it. So for example 0+1=1, 1+1=2, 1+2=3, 2+3=5, and so on.


Implementation

The program I wrote is in MIPS assembly language and is being run in MARS(MIPS and Runtime Simulator). It is given a number, and uses recursion to find the Nth number in the Fibonacci Sequence where N is the given number.


Importance

The significance of this project comes from the demonstration of understanding of very low-level coding. I love the low-level parts of computer science and the class this project comes from is all about those. Without these parts everything built on top would not exist.

The special part about this specific program that makes it one of my highlights is that the professor pulled it up and used it as an example of an excellent solution. The fact that my solution was so highly praised made it very special to me, especially because of the class it was in and what it represents understanding of.

SHA3-ECDHIES Cryptography Application


Description

Cybersecurity is a very important part of life on the internet. With so much data about us on computers, there is an opportunity for that data to be viewed, changed, corrupted, or otherwise interacted with by people who have bad intentions or just shouldn't have the ability to do so. Through the basic cybersecurity services confidentiality, integrity, authentication(legitimacy, non-repudiation), and availability(this one cannot be done with cryptography) we assure that our data stays safe.


Implementation

In this project, I implemented a program which allows for hashing, encrypting, decrypting, tagging, signing, and verifying data. The SHA3 derived KMACXOF256 function is used prominently in this project. For encrypting and decrypting, there are options to use both the symmetric algorithms and elliptic curve algorithms. The primary method of passing data in is through files although there are text input functionalities available. If I were to continue development of this project I would make it more robust and user-friendly, as the focus of the project was the functionality of the cryptography.


Importance

To me this project was very important because as I said in the Who I Am section of the home page, I am driven by the need to know how things work. This project was very special to me because it allowed me to implement the standardized cryptography functions by myself and at the end I had a program that I wrote which could perform the basic cryptographic cybersecurity services. I'm very proud of this because that means that in the future if I want to ensure the security of something I'm working on, I've proven that I can do that myself and I don't have to rely on someone else to do that for me. At the very least it means I understand how the concepts work and makes me feel more secure in my future work, no matter the focus. I also got a lot of practice with software development, implementing things based on standards, working between different languages, and source control.
